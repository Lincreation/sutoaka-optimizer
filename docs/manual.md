# PART 1：はじめに（全員向け）

## このツールは何？

ストアカ講座割り振りツールは、**毎月の講座予約作業を担当者に自動で割り振り**、**Slack通知用のメッセージデータを生成する**Webアプリケーションです。

- ブラウザだけで動作（インストール不要）
- アカウント登録・ログイン不要
- データはブラウザ内に自動保存

## 解決する課題

毎月、複数の講座に対して30名以上の担当者に「誰が」「いつ」「どの講座を」「何回」予約するかを決める作業があります。手作業では以下の問題が発生していました：

| 課題 | 内容 |
|------|------|
| 割り振りの偏り | 特定の人に作業が集中、月の前半に偏る |
| 調整コスト | Excelで手動調整すると数時間かかる |
| 通知作成の手間 | 一人ひとりにSlackメッセージを作成する必要 |

**→ このツールは、ボタン一つで最適な割り振りを自動生成し、Slack通知用データまで一括出力します。**

## アクセス方法

ブラウザで以下のURLを開くだけです：

## 初回セットアップ

初回アクセス時には、あらかじめ登録された **13講座・5グループ・31名の担当者データ** が自動で読み込まれます。
以下の手順で内容を確認・調整してください。

### ステップ1：期間を設定する

1. サイドバーから「**設定**」を開く
2. 「**期間**」タブを選択
3. **開始日**と**終了日**を設定（通常は当月の1日〜末日）

### ステップ2：講座を確認する

「**講座**」タブで各講座の情報を確認します：

| 項目 | 説明 |
|------|------|
| 講座名 | 表示名（例：「竹内_占い」） |
| URL | ストアカの講座ページURL |
| 目標回数 | 月間の目標予約数 |
| 1日回数 | 1日に入れる最大回数（空欄=デフォルト2回） |

- 表の中を直接クリックして編集可能
- 「**+ 講座を追加**」で新規追加
- **☰** マークをドラッグで並び替え

### ステップ3：グループを確認する

「**グループ**」タブで各グループの担当講座を確認します。チェックボックスで担当可能な講座を選択できます。

### ステップ4：担当者を確認する

「**担当者**」タブで各メンバーを確認します：

| 項目 | 説明 |
|------|------|
| 名前 | 担当者の氏名 |
| グループ | 所属グループ（ドロップダウンで選択） |
| SlackユーザーID | Slackのメンション用ID（例：U09D156EG1J） |
| 月上限 | 月間の最大担当回数（空欄=無制限） |
| 有効 | チェックON=割り当て対象、OFF=対象外 |

> 💡 退会・休止中のメンバーは「有効」チェックを外してください

### ステップ5：ルールを確認する

「**ルール**」タブで割り振りの制約を設定します：

| 設定 | 標準値 | 説明 |
|------|--------|------|
| 1人1日あたり合計上限 | 5回 | 1人が1日に担当する最大予約回数 |
| 週あたり稼働日数 | 3日 | 1人が1週間に働く最大日数 |

## 割り当てを実行する

### 実行手順

1. サイドバーから「**割り当て実行**」を開く
2. 「実行前チェック」で期間・講座数・担当者数を確認
3. **「割り当て実行」ボタンをクリック**
4. 数秒で結果が表示される

### 結果の見方

| 表示 | 意味 |
|------|------|
| 合計割り当て回数 | 全講座の合計予約回数 |
| 目標達成率 | 100%以上なら全目標達成 |
| 担当者数 | 割り当てられたメンバー数 |
| 稼働日数 | 使用された日数 |

### 診断結果について

| レベル | 意味 |
|--------|------|
| ✅ OK | 正常（目標達成） |
| ⚠️ 注意 | 担当者間の偏り（3回以内の差は許容範囲） |
| ❌ エラー | 設定を見直して再実行が必要 |

> やり直す場合は「**再実行**」ボタンを押すだけです

## 結果を確認する

### ダッシュボード
全体サマリー。講座別の達成率、担当者別の負荷、グループ別のバランスを一覧表示。

### 日付別ビュー

メンバーごとのスケジュール。**テーブル表示**と**カレンダー表示**の2モードあり。カレンダーでは月間の稼働パターンが一目でわかります。**◀▶ボタン**で別の月のカレンダーに切り替え可能。

### 講座別ビュー
各講座について、どの日に誰が担当するかを**カレンダー**または**テーブル**で確認。カレンダーは**◀▶ボタン**で月を切り替えられます。

## 通知データを出力する

### メッセージ形式

各担当者への通知メッセージは以下の形式で生成されます：

```
<@U09D156EG1J>
お世話になっております。
本日、以下の講座に予約してください。
・新井_AI副業（https://www.street-academy.com/...）：2回
・林_AIライティング（https://www.street-academy.com/...）：2回
予約完了したらスレッドに報告お願いします！
```

> `<@U09D156EG1J>` の部分が、Slackに貼り付けると自動でメンション（通知）になります

### エクスポート方法

| ボタン | 用途 |
|--------|------|
| 📋 スプシ用コピー（TSV） | Googleスプレッドシートに貼り付け（Ctrl+V）。改行を保持 |
| 📊 Excel（グループ別シート） | **1つの.xlsxファイル**にグループ別タブで出力。「全体」シート＋各グループのシートが含まれる |
| 📥 CSVダウンロード | 表示中のデータをCSVファイルとしてダウンロード |
| 📦 グループ別CSV一括DL | 全体CSV＋グループごとのCSVを個別ファイルで一括ダウンロード |
| 💾 TSVダウンロード | TSVファイルとしてダウンロード |
| 📝 表示中を全てコピー | プレーンテキストとして全メッセージコピー |

各通知カードの右上「**コピー**」ボタンで、一人分だけコピーすることもできます。

> 💡 **おすすめ**: 「📊 Excel（グループ別シート）」を使うと、スプレッドシートのタブ分けと同じように1ファイルでグループ別データを管理できます。Googleスプレッドシートにインポートすればそのままタブで切り替え可能です。

### テンプレートのカスタマイズ

「設定」→「テンプレート」タブで冒頭文と締め文を自由に編集できます。プレビューでリアルタイムに確認可能。

## よくある質問

### Q. データが消えた場合は？
ブラウザのキャッシュをクリアするとデータがリセットされますが、**講座・グループ・担当者・Slack IDなどの基本データはデフォルト値として自動復元**されます。割り当て結果のみ再実行が必要です。

### Q. 別のPCで使いたい場合は？
URLにアクセスすれば同じツールが使えます。**デフォルトデータはコードに含まれている**ため、どのPCでも同じ初期データが表示されます。ただし、設定の変更は各ブラウザ独立です。

### Q. 担当者が増えた・減った場合は？
「設定」→「担当者」タブで追加・削除・有効無効の切り替えが可能。変更後に「割り当て実行」を再実行してください。

## 割り当てアルゴリズム詳細

`allocationEngine.ts` に実装。6段階で処理：

### 第1段階：バリデーション
期間・講座・グループ・メンバーの整合性チェック

### 第2段階：スロットベースのクォータ配分
```
スロット数 = ceil(目標回数 / perDay)
メンバーあたりスロット = スロット数 / メンバー数（均等配分）
メンバーあたりクォータ = メンバーあたりスロット × perDay
```
端数処理による過剰割り当てを最小化する方式。

### 第3段階：ビンパッキングによる必要日数計算
`computeMinDaysBinPack` 関数（FFD: First-Fit-Decreasing）：
- 各「ビン」= 1日（容量 = maxPerDay）
- 各「アイテム」= 講座の1日分スロット（サイズ = perDay）
- **制約**: 同一講座が同一日に2回入れない
- 例: perDay=3の講座が2つ → 3+3=6 > maxPerDay=5 → 同じ日に入れられない → 2日必要

### 第4段階：週間分散配置
- `mergeShortEdgeBuckets`: 3日未満の端数週を隣の週にマージ
- 週ごとに比例配分 + モジュラーステップで日をずらし、メンバー間で同一日集中を回避

### 第5段階：講座パッキング（距離最大化 + メンバー別オフセット）
既に使用中の日からの距離が最大の日を選択し、月全体に均等分散。各講座の初期配置位置を `memberIndex` でオフセットすることで、同じ講座を複数メンバーが担当する場合でも月内で日付が分散される。

### 第6段階：拡張ループ
日数不足時に追加日を選択。距離最大化ヒューリスティックにより月初偏りを防止。

# PART 4：開発プロセス

## 開発の経緯

### 目的
ストアカ（street-academy.com）の講座予約を担当者に割り振る作業の自動化。以前はExcelで手動管理していたが、30名以上×13講座の組み合わせで月初に数時間を要していた。

### 技術選定の理由

| 判断 | 理由 |
|------|------|
| SPA（クライアントサイドのみ） | サーバーコストゼロ、認証不要、即利用開始可能 |
| localStorage | サーバーDB不要、オフラインでも動作 |
| GitHub Pages | 無料ホスティング、社内展開のハードル最小 |
| 最小限の外部ライブラリ | SheetJS(xlsx生成)のみ追加。ビルドサイズを抑制 |

### アルゴリズムの進化

**初期実装の問題：**
- 単純な `目標回数 / maxPerDay` で日数算出 → perDay制約の衝突を考慮できず日数を過小評価
- 日付を先頭から順に埋める → 3/1に22名、3/2に18名と月初に集中

**改善（ビンパッキング + 距離最大化）：**
- FFDビンパッキングで正確な必要日数を計算
- 拡張ループで距離最大化ヒューリスティックを適用
- **結果: 1日あたり担当者数が 2〜22名 → 4〜8名に改善**

### データ永続化の設計判断

開発中にlocalStorageクリアにより、入力済みの講座URL（13件）やSlack ID（31件）が消失するインシデントが発生。この教訓から、**重要な初期データはすべてコード内のデフォルト値として埋め込む設計**に変更。URLを共有するだけで誰でも同じ初期データでツールを利用開始可能。

## 主要な設計パターン

| パターン | 適用箇所 |
|----------|----------|
| 状態ベースルーティング | App.tsx（React Router不使用） |
| Context + Hooks | 状態管理（useAppState） |
| 自動永続化 | useLocalStorage → localStorage |
| HTML+TSV二重コピー | Sheets互換エクスポート |
| UTF-8 BOM付きCSV/TSV | Excel/Sheets日本語文字化け防止 |
| SheetJS xlsx出力 | グループ別シートを1ファイルに統合 |
| 設定CSV入出力 | 通知データ・設定のインポート/エクスポート |
